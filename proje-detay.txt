## 5. YAPILANDIRILMIŞ VERİ (JSON-LD / Schema.org)

Her ürün sayfasında `<script type="application/ld+json">` içinde **Product schema** bulunuyor. Bu, en temiz veri kaynağı:

```json
{
  "@type": "Product",
  "name": "Ürün Adı",
  "sku": "153418",                 ← VATANBİLGİSAYAR İÇ ID
  "mpn": "SM-S731BZKGTUR",         ← ÜRETICI PARÇA NO
  "brand": { "name": "SAMSUNG" },
  "color": "Siyah",
  "category": "Tüketici Elektroniği > Cep Telefonu > ...",
  "offers": {
    "priceCurrency": "TRY",
    "price": "35499",              ← KURUŞ YOK, NOKTA YOK, SAF SAYI
    "availability": "https://schema.org/InStock",
    "priceValidUntil": "2027-04-05"
  }
}
```

**Tavsiye:** Bot, ürün detay sayfalarından JSON-LD parse ederek fiyat çekerse hem en temiz hem de en stabil yöntemdir.

## 8. ANTI-BOT / RATE LIMIT NOTLARI

- Site Cloudflare kullanıyor → çok hızlı istek atılırsa 403/503 dönebilir
- `robots.txt` kontrol edilmeli
- Makul bekleme süresi: istekler arası **2-5 saniye** önerilir
- User-Agent değiştirme gerekebilir
- Görüntü lazy-load kullanıyor (fiyatlar HTML'de mevcut, JS render gerektirmiyor)

  Sayfalama: ?page=2, ?page=3 (her sayfada 24 ürün)

FİYAT VERİSİ ÇEKME YÖNTEMİ (2 seçenek, öncelik sırasıyla):

Yöntem 1 - JSON-LD (Tavsiye Edilen):
Her ürün sayfasında <script type="application/ld+json"> etiketi içinde
Product schema var. "offers.price" alanı KURUŞsuz, noktas1z tam sayı verir.
Örnek: {"@type":"Product","sku":"153418","offers":{"price":"35499","priceCurrency":"TRY","availability":"https://schema.org/InStock"}}

Yöntem 2 - CSS Selektörler:

  - SKU kodu:      .product-list__product-code
  - Fiyat (rakam): .price-basket-camp--new-price .product-list__price
  - Eski fiyat:    .price-basket-camp--new-price .product-list__current-price
  - Ürün linki:    a.product-list-link (href attribute)

Ürün detay sayfası:
  - Başlık:        h1
  - Fiyat büyük:   .product-detail-price-big
  - SKU:           .product-list__product-code (format: "MPN / İÇ_ID")
  - Stok:          offers.availability JSON-LD'den ("InStock"/"OutOfStock")
  - Puan:          .wrapper-star

== VERİTABANI ŞEMASI ==

CREATE TABLE products (
  id INTEGER PRIMARY KEY,
  sku TEXT NOT NULL,          -- Vatan iç ID (örn: 153418)
  mpn TEXT,                   -- Üretici kodu (örn: SM-S731BZKGTUR)
  name TEXT,
  url TEXT UNIQUE,
  brand TEXT,
  category TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE price_history (
  id INTEGER PRIMARY KEY,
  product_sku TEXT,
  price REAL,                 -- TRY cinsinden
  old_price REAL,             -- Varsa çizili eski fiyat
  in_stock BOOLEAN,
  scraped_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (product_sku) REFERENCES products(sku)
);

CREATE TABLE alerts (
  id INTEGER PRIMARY KEY,
  product_sku TEXT,
  target_price REAL,          -- Kullanıcının hedef fiyatı
  alert_sent BOOLEAN DEFAULT FALSE,
  created_at DATETIME
);

== BOT MİMARİSİ ==

Dosya yapısı:
  vatan_bot/
  ├── config.py          # Ayarlar (API key, hedef URL'ler, eşik değerleri)
  ├── scraper.py         # Ana web scraping modülü
  ├── db.py              # SQLite veritabanı işlemleri
  ├── notifier.py        # Telegram/e-posta bildirim
  ├── scheduler.py       # APScheduler ile zamanlama
  ├── main.py            # Giriş noktası
  └── requirements.txt

Kütüphaneler:
  - requests + BeautifulSoup4 (scraping)
  - sqlite3 (veritabanı)
  - APScheduler (zamanlama)
  - python-telegram-bot veya smtplib (bildirim)
  - fake-useragent (bot tespitinden kaçınmak için)

== SCRAPING KURALLARI ==

1. İstekler arası 2-5 saniye rastgele bekleme ekle
2. User-Agent'ı tarayıcı gibi ayarla (fake-useragent kullan)
3. HTTP 429 veya 503 alınca exponential backoff uygula (1s → 2s → 4s → 8s)
4. Session kullan, cookies sakla
5. Cloudflare challenge sayfası gelirse 60 saniye bekle ve tekrar dene

Headers örneği:
{
  "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) ...",
  "Accept-Language": "tr-TR,tr;q=0.9",
  "Accept": "text/html,application/xhtml+xml,...",
  "Referer": "https://www.vatanbilgisayar.com/"
}

== FIRSATI TESPİT ETme KURALLARI ==

Fırsat türü 1 - Fiyat Düşüşü:
  Yeni fiyat < Son kaydedilen fiyat × (1 - THRESHOLD)
  Örnek THRESHOLD = 0.05 → %5'ten fazla düşüş = fırsat

Fırsat türü 2 - Hedef Fiyata Ulaşma:
  Güncel fiyat ≤ alerts tablosundaki target_price

Fırsat türü 3 - Fırsat Sayfasında Yeni Ürün:
  /firsat-urunler sayfasında daha önce görülmemiş ürün

Fırsat türü 4 - Tarihsel En Düşük Fiyat:
  Güncel fiyat = MIN(price_history.price) for this SKU

## 10. ÖZET: YAPMANIZ GEREKENLER

Adım adım yapılacaklar şunlardır:

**1. Altyapı kurulumu:** Yukarıdaki prompt'u bir AI kodlama ajanına (Claude, GPT-4, Cursor vb.) verin, kodu ürettirin.

**2. Telegram botu oluşturun:** `@BotFather`'dan yeni bot açıp API token alın. Bu token'ı `config.py`'ye koyun.

**3. Takip listesi hazırlayın:** Takip etmek istediğiniz ürünlerin `.html` URL'lerini toplayın ve `products` tablosuna ekleyin.

**4. Sunucuya taşıyın:** Bot sürekli çalışması için VPS veya bir Raspberry Pi üzerinde çalıştırın.

**5. Bildirim eşiğini ayarlayın:** `THRESHOLD = 0.05` ile başlayın, çok gürültülü olursa `0.10`'a çıkarın.

En kritik teknik nokta şudur: Fiyatları JSON-LD'den parse etmek (Yöntem 1) CSS selector'lara göre çok daha stabil çalışır; site tasarımı değişse bile JSON-LD genellikle aynı kalır.
